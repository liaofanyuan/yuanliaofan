产品开发文档（PRD）

产品名称（暂定）

《了凡四训 · 自主学习模式（Active Reading）》

⸻

一、产品背景与目标

1. 背景

当前 yuanliaofan.com 的内容形态为：
	•	📖 单页连续原文展示
	•	✅ 适合“通读”“查阅”
	•	❌ 不利于：
	•	结构化理解
	•	主动思考
	•	反复复习
	•	学习成效验证

而《立命之学》这类文本，本身具备：
	•	清晰的论证结构
	•	明确的转折点
	•	可被拆解的思想模块
	•	非常适合 “模块化 + 提问式阅读”

2. 产品目标

将「被动阅读古文」升级为「结构引导 + 思维反馈 + 学习验证」的自主学习体验。

具体目标：
	1.	帮助用户看清文章脉络
	2.	强化“读 → 想 → 验证”的学习闭环
	3.	不改变原文，只改变呈现与交互方式
	4.	保持简洁、克制、可长期维护

⸻

二、用户画像与使用场景

目标用户
	•	对《了凡四训》有长期阅读/修行/思考需求的人
	•	已有一定阅读能力，但希望理解得更深
	•	偏向个人学习，而非娱乐型用户

使用场景
	•	每天阅读一个模块（5–15 分钟）
	•	碎片时间 + 深度阅读混合
	•	反复回看结构总结
	•	阅读完成后进行自测，确认理解是否偏差

⸻

三、核心设计理念

1️⃣ 内容不变，结构重组

原文是“线性文本”，学习是“分层认知”。

2️⃣ 阅读节奏由系统引导

一段一段读，而不是一口气读完。

3️⃣ 强制“停下来想一下”

通过按钮 / 卡片 / 小测试，让用户从阅读切换到思考状态。

⸻

五、章节内结构设计（以〈立命之学〉为例）

模块拆分（Module-based）

将《立命之学》拆分为 6 个学习模块（与你之前的脉络完全一致）：

模块	模块名称	颜色建议
M1	宿命论的建立	灰蓝
M2	云谷禅师的否定	深蓝
M3	命由我作的理论	青绿
M4	自我剖析：为何不配	橙色
M5	改命的方法论	深绿
M6	结果验证与总结	紫色

颜色作用：结构标识，而非装饰

⸻

六、单个模块的页面结构（核心）

模块页面布局

┌────────────────────┐
│ 模块标题 + 进度条   │
├────────────────────┤
│ 原文内容（限定段落）│
│                    │
│ （无干扰，纯阅读） │
├────────────────────┤
│ 【查看本模块大纲】 │
│ 【我已理解，进入下一模块】 │
└────────────────────┘


⸻

1️⃣ 原文呈现
	•	只展示当前模块对应的原文段落
	•	禁止滚动看到下一个模块
	•	保留你现有的字体/配色系统

⸻

2️⃣「查看本模块大纲」按钮（关键功能）

点击后，以 Overlay / 抽屉 / 卡片 形式展示：

示例（M1）：
本模块要点总结
	•	作者先完整展示“命被算定”的人生样本
	•	多次应验不是为了证明算命
	•	而是为后文“破命论”做铺垫

可选增强：
	•	「一句话总结」
	•	「本模块你需要记住的是什么」

⸻

3️⃣ 模块完成机制
	•	点击 「我已理解，进入下一模块」
	•	自动：
	•	标记模块完成
	•	解锁下一个模块
	•	更新顶部进度条

⸻

七、通读完成后的「学习测试模块」

测试触发条件
	•	六个模块全部完成后自动解锁

⸻

测试形式
	•	单选题
	•	每题 4 个选项
	•	不计分数排名
	•	目的：自我校准理解是否偏差

⸻

测试题内容来源

👉 直接使用我之前给你的那组选择题

结构示例：

( ) A
( ) B

( ) D

【提交本题】


⸻

答题反馈设计（重要）
	•	选择后立即反馈：
	•	✅ 正确：简短解释
	•	❌ 错误：指出“误解点在哪里”
	•	不展示“你击败了多少人”

⸻

八、学习完成页（Closing）

完成测试后，展示一个总结页：

内容包括：
	•	本章核心结论（一句话）
	•	你的测试正确率（仅自己可见）
	•	可选入口：
	•	🔁 重新学习某一模块
	•	➡️ 进入下一章《改过之法》

⸻

九、技术实现建议（轻量优先）

前端
	•	Vue / React 均可
	•	模块结构 JSON 化（可复用到其它章节）
	•	颜色、模块标题、段落索引配置化

数据（本地即可）
	•	localStorage 记录：
	•	模块完成状态
	•	测试结果（可选）

❗️无需账号系统，降低维护成本

	•	学习笔记导出
